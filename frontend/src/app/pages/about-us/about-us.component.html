About This Project

This project is aimed at building a lexical analyzer, consisting of two main components: a scanner and an SLR table analyzer. The scanner reads the text and transforms it into tokens, which are then evaluated by the SLR table.

Scanner

The scanner component is responsible for reading the input text and breaking it down into tokens. It is based on YAL (Yet Another Lexer) files, which define the lexical rules. Here's an example of a YAL file: 


(* Lexer for Grammar No. 1 - Simple arithmetic expressions for variables *)

(* Insert any headers here *)

let delim = [' ' '\t' '\n']
let ws = delim+
let letter = ['A'-'Z' 'a'-'z']
let digit = ['0'-'9']
let id = letter (letter | digit)*

rule tokens =
    ws        { return WS }
  | id        { return ID }               (* Replace with a valid action that returns the token *)
  | '+'       { return PLUS }
  | '*'       { return TIMES }
  | '('       { return LPAREN }
  | ')'       { return RPAREN }

(* Insert any trailers here *)

In this example, various patterns are defined for whitespace (ws), letters (letter), digits (digit), and identifiers (id). The tokens rule specifies how different patterns should be recognized and what tokens they should return.

SLR Table

The SLR (Simple LR) parser table is used to evaluate the tokens generated by the scanner. The parser configuration is based on an SLR grammar, which defines the parsing rules. Here is an example of an SLR parser configuration: 


/* Parser configuration for Grammar No.1 */
/* Follows the SLR grammar: */
/* E -> E + T | T */
/* T -> T * F | F */
/* F -> ( E ) | id */

%token ID
%token PLUS
%token TIMES
%token LPAREN RPAREN
%token WS
IGNORE WS

expression:
    expression PLUS term
  | term
;
term:
    term TIMES factor
  | factor
;
factor:
    LPAREN expression RPAREN
  | ID
;

This configuration defines tokens such as ID, PLUS, TIMES, LPAREN, and RPAREN. It also specifies rules for expressions, terms, and factors, describing how they can be composed.

What is a Scanner?

A scanner, also known as a lexical analyzer, is a program that reads the source code and converts it into a sequence of tokens. Tokens are the smallest units of meaning in the source code, such as keywords, identifiers, operators, and punctuation. The scanner uses lexical rules, often defined in a lexer file, to identify and categorize these tokens.

What are SLR Tables?

SLR (Simple LR) tables are used in parser generators for syntax analysis. An SLR parser uses a parsing table to determine the actions to take (shift, reduce, accept, or error) based on the current state and the lookahead token. The parsing table is constructed from the grammar rules and is used to efficiently parse the input tokens and build the syntax tree.

The combination of the scanner and the SLR parser allows the system to read input text, tokenize it, and then parse the tokens according to the specified grammar rules, enabling the analysis and processing of programming languages and other structured text.
